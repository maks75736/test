# 📘 Повний гайд: Встановлення MkDocs та деплой на GitHub Pages

Цей посібник містить усі необхідні команди та кроки для створення сучасного сайту документації, вирішення проблем із терміналом та успішної публікації в мережі.

## 1. Встановлення та запуск MkDocs (в терміналі)

* `pip install mkdocs` (або `pip3 install mkdocs`) — для встановлення бібліотеки MkDocs.
* `mkdocs new my_docs` — для створення нового проекту з документацією (де `my_docs` — це назва вашої папки).
* `mkdocs serve` — для локального запуску сервера. Дозволяє переглядати зміни в реальному часі за адресою `http://127.0.0.1:8000`.

---

## 2. Команди Git для ініціалізації та першого коміту

* `git init` — ініціалізація локального Git-репозиторію в папці проекту.
* `git remote add origin <посилання_на_репозиторій>` — підключення локального репозиторію до вашого проекту на GitHub/GitLab.
* `git status` — перевірка статусу файлів перед додаванням.
* `git add .` — додавання всіх файлів до індексу.
* `git commit -m "commit inicial"` — створення першого коміту.
* `git push origin master` (або `main`) — відправка коду на сервер.

---

## 3. Команди всередині файлу `.gitlab-ci.yml` (для CI/CD)

Якщо ви використовуєте GitLab, у скриптах автоматизації зазвичай прописуються такі команди:
* `pip install mkdocs==<версія>` (або `pip install -r requirements.txt`) — встановлення потрібної версії перед збиранням.
* `mkdocs build` — генерація статичних HTML-файлів сайту.

---

## 4. Цикл збереження та оновлення змін

Після внесення будь-яких змін (додавання нових сторінок або редагування конфігу), використовуйте цей цикл:
1. `git add .` — підготовка нових файлів.
2. `git commit -m "ваш коментар"` — збереження версії.
3. `git push origin main` — відправка коду в основну гілку.

---

## 💡 Швидкий фікс (якщо MkDocs не розпізнається)

Якщо система видає помилку, що команду `mkdocs` не знайдено, звертайтеся до неї безпосередньо через Python:

* Замість `mkdocs new my_docs` пишіть: **`python -m mkdocs new my_docs`**
* Замість `mkdocs serve` пишіть: **`python -m mkdocs serve`**
* Замість `mkdocs gh-deploy` пишіть: **`python -m mkdocs gh-deploy`**

## 🚀 Покрокова інструкція: Деплой на GitHub Pages

### Крок 1: Завантажте проект на GitHub
```bash
git init
git add .
git commit -m "Перший коміт документації"
git remote add origin https://github.com/ВАШЕ_ІМ'Я/ВАШ_РЕПОЗИТОРІЙ.git
git push -u origin main
```

### Крок 2: Команда gh-deploy
MkDocs має вбудовану функцію, яка створює гілку `gh-pages` і збирає туди сайт:
```bash
python -m mkdocs gh-deploy
```
*(Це згенерує папку `site` та автоматично відправить її у відповідну гілку на GitHub).*

### Крок 3: Налаштування GitHub Pages
1. Зайдіть у свій репозиторій на GitHub -> **Settings**.
2. Зліва оберіть **Pages**.
3. У розділі "Build and deployment" встановіть:
   * **Source**: Deploy from a branch.
   * **Branch**: `gh-pages` та папка `/(root)`.
4. Натисніть **Save**.

> **Важливо:** Ваш репозиторій має бути **публічним (Public)**, щоб GitHub Pages працював безкоштовно.

## ✨ Порада щодо оформлення (Material Theme)

Щоб сайт виглядав професійно, встановіть тему **Material**:

1. Встановіть її через термінал: `pip install mkdocs-material`.
2. У файлі `mkdocs.yml` змініть налаштування:
```yaml
theme:
  name: material
```

---

## 🔄 Як опублікувати нові зміни
Кожного разу, коли ви змінюєте вміст (наприклад, додали новий гайд), робіть наступне:

**1. Оновіть вихідний код:**
```bash
git add .
git commit -m "Оновив контент"
git push origin main
```

**2. Оновіть сам сайт в інтернеті:**
```bash
python -m mkdocs gh-deploy
```
