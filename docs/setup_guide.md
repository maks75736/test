**1. Встановлення та запуск MkDocs (в терміналі):**

- `pip install mkdocs` (або `pip3 install mkdocs`) — для встановлення бібліотеки MkDocs. 
- `mkdocs new mkdocs` — для створення нового проекту з документацією (де друге "mkdocs" — це назва папки проекту). 
- `mkdocs serve` — для локального запуску сервера, щоб переглядати зміни в документації у реальному часі (доступ зазвичай на порту 8000). 
    
**2. Команди Git для ініціалізації та першого коміту:**
 
- `git init` — ініціалізація локального Git-репозиторію в папці проекту.
- `git remote add origin <посилання_на_репозиторій>` — підключення локального репозиторію до створеного на GitLab. 
- `git status` — перевірка статусу файлів перед додаванням. 
- `git add .` — додавання всіх невідстежуваних файлів до індексу.
- `git commit -m "commit inicial"` — створення першого коміту. 
- `git push origin master` — відправка коду на віддалений сервер GitLab. 
 
**3. Команди, прописані всередині файлу `.gitlab-ci.yml` (для CI/CD):** У скриптах автоматизації для GitLab Pages автор вказував такі команди терміналу, які мають виконатись на сервері:

- `pip install mkdocs==<версія>` (або `pip install -r requirements.txt`) — для встановлення правильної версії MkDocs перед збиранням. 
- `mkdocs build` — для генерації статичних HTML-файлів сайту на основі написаної документації.
**4. Наступні команди Git для збереження змін:** Після внесення змін (додавання файлу `.gitlab-ci.yml` та оновлення структури сайту), автор використовував звичний цикл команд для збереження:
- `git commit -m "ваш кометар"` — коміт зі збереженням налаштувань CI/CD.
- `git push origin master` — відправка оновлень. 
- `git add .` — додавання нових markdown-файлів документації. 
- `git commit -m "ваш коментар"` — коміт зі створеною структурою.
- `git push origin master` — фінальна відправка готової документації на GitLab.

### Швидкий фікс
Замість просто `mkdocs`, звернись до нього через Python. Це майже завжди спрацьовує, бо сам `python` система бачить:
- Замість `mkdocs new my_docs` напиши: `python -m mkdocs new my_docs`
- Замість `mkdocs serve` напиши: `python -m mkdocs serve`


## покрокова інструкція, як перенести вашу документацію на GitHub Pages:

### Крок 1: Завантажте проект на GitHub

Якщо ви ще цього не зробили, створіть репозиторій на GitHub і відправте туди свій код:

Bash

```
git init
git add .
git commit -m "Перший коміт документації"
git remote add origin https://github.com/ВАШЕ_ІМ'Я/ВАШ_РЕПОЗИТОРІЙ.git
git push -u origin main
```

---

### Крок 2: Найпростіший спосіб — команда `gh-deploy`

MkDocs має спеціальну команду, яка сама створює окрему гілку (`gh-pages`), збирає туди статичний сайт і завантажує його на GitHub.

У терміналі (всередині папки проекту) введіть:

Bash

```
python -m mkdocs gh-deploy
```

_(Якщо ви вже налаштували PATH, можна просто `mkdocs gh-deploy`)_.

**Що відбудеться:**

1. MkDocs згенерує папку `site`.
    
2. Створить у вашому репозиторії на GitHub нову гілку з назвою **gh-pages**.
    
3. Скопіює туди сайт і «запушить» його.
    

---

### Крок 3: Увімкніть GitHub Pages у налаштуваннях

Після того, як команда завершить роботу:

1. Зайдіть у свій репозиторій на сайті GitHub.
    
2. Перейдіть у вкладку **Settings** (Налаштування).
    
3. Зліва в меню оберіть **Pages**.
    
4. У розділі "Build and deployment" переконайтеся, що:
    
    - Source: **Deploy from a branch**.
        
    - Branch: обрано **gh-pages** та папка **/(root)**.
        
5. Натисніть **Save**.
    

Через 1-2 хвилини зверху з'явиться посилання на ваш сайт (наприклад, `https://username.github.io/repo-name/`).

### **ще одне уточнення ваш репозиторій має бути публічним**
### Порада щодо оформлення

Якщо ви хочете, щоб сайт виглядав сучасно (як у більшості IT-проектів), у файлі `mkdocs.yml` змініть тему на **material**:

1. Встановіть її: `pip install mkdocs-material`.
    
2. У `mkdocs.yml` напишіть:
    
    YAML
    
    ```
    theme:
      name: material
    ```
### Як опублікувати зміни

	Після того, як ви відредагуєте ці файли у себе на комп'ютері, виконайте стандартні команди для оновлення сайту на GitHub Pages:

1. **Збережіть зміни в основну гілку:**
    
    Bash
    
    ```
    git add .
    git commit -m "Додав головну сторінку та посилання на гайд"
    git push origin main
    ```
    
2. **Запустіть деплой сайту:**
    
    Bash
    
    ```
    python -m mkdocs gh-deploy
    ```s
